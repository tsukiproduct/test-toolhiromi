AI SIDE LAB / EDITOR OS

使い方ガイド（プレビュー版）

このツールは「AIニュースを、noteに投稿できる記事の材料に変える」ための編集作業台です。ニュースを読むためのサイトではなく、記事を量産するための道具です。

プレビュー版でできること・できないこと

  • できるニュースを選んで、記事ネタ・設計書・プロンプト・タイトル案・SNS投稿文を見る／コピーする
  • できるプロンプトをClaude等に貼って、記事本文を実際に生成する（記事化はここで完結）
  • できる1つのニュースから無料記事〜有料記事まで、複数の記事を作る（下の育成ロードマップ参照）
  • できない本物の最新ニュースの表示 — プレビュー版は練習用のデモニュース8件で固定です。本物を流すにはGitHubへの公開（push）が必要
  • できない自動更新 — デモデータなので何度開いても同じ8件です
  • できないnoteへの自動投稿 — noteに公式の投稿APIがないため、本番版でも「コピペ投稿」が仕様です（これは制限ではなく安全策）
  • 未実装記事ネタとプロンプトの連動 — 現状は汎用プロンプトのため、角度を変えたいときはプロンプト冒頭に「初心者向けの角度で」等を一言足してください（連動機能は追加開発で対応可能）

基本の使い方（5ステップ）

1

ニュースを1つタップ

一覧のカードの「タイトル文字」をタップ。note/X/YT…の数字はスコア表示で、押す場所ではありません。

2

「記事ネタ」で角度を決める

初心者向け／副業／比較／体験談／収益化の5案から「今回はこれで書く」と決めるだけ。

3

「プロンプト」タブへ

使うLLMの枠（Claudeなど）の「コピー」を1つ押す。4つ全部は不要です。

4

LLMに貼って送信

Claudeアプリ等を開き、入力欄を長押し→貼り付け→送信。2,600〜3,000字の記事ドラフトが出ます。

5

noteに貼って投稿

出てきた本文をnoteの編集画面にコピペ。タイトルは「タイトル20」から、告知文は「SNS」タブから選べます。

投稿前に必ず一読を。LLMは数字や固有名詞を間違えることがあります。「出典を読む ↗」の元記事と食い違いがないかだけ確認してから投稿してください。

タブの役割（タップ後の画面）

記事ネタ

このニュースから書ける記事5案。角度（誰向けに書くか）を選ぶ場所。

note設計書

記事の骨組み（読者・悩み・構成・SEO）。読むだけでOK。中身はプロンプトに全部入っています。

プロンプト

記事化のエンジン。LLM別に4種。コピーして貼るだけ。

タイトル20

記事タイトルの候補20個。気に入ったものを選ぶか、組み合わせて使う。

SNS

投稿後の告知文。X／Threads／Instagram用。（記事URL）の部分だけ差し替え。

ロードマップ

このニュースを無料→有料へ育てる展開計画。詳細は次の項目へ。

1つのニュースから複数の記事を作る

このツールの核です。同じニュースでも「角度」を変えれば別の記事として成立します。無料記事で読者を集め、信頼が溜まったら有料へ——という流れを1ニュースで設計できます。

LV.1速報ポストX / 当日
LV.2初心者向け解説note無料 / 1〜2日後
LV.3比較記事note無料 / 3〜4日後
LV.4体験レビューnote無料+X / 1週間後
LV.5収益化記事note有料 / 2週間後
LV.6テンプレ集など商品化Brain・note有料 / 1ヶ月後

使用例：デモの「生成AIの副業活用」ニュースの場合

月曜　記事ネタ「初心者向け」→ プロンプトをClaudeに貼る → 「生成AIの副業活用とは？3分で解説」をnote無料で投稿。SNSタブのX文で告知 水曜　同じニュースで角度を「比較」に。プロンプト冒頭に「比較記事の角度で」と一言足して生成 → note無料で2本目 翌週　自分で試した感想を混ぜて「体験レビュー」→ 3本目。ここまでで読者が育つ 2週間後　「収益化」角度で具体的ワークフロー記事 → note有料（例:300〜500円）。無料3本が営業マンになってくれる

つまり、ニュース1件 = 記事1本ではなく、ニュース1件 = 無料3〜4本 + 有料1〜2本の「シリーズ」として使うのが本来の設計です。

スコアの見方

カードの下の数字は「このニュースがどの場所に向いているか」の適性（0〜99）です。

note 97 → note記事にすると伸びやすい題材
X 96 → 速報ポスト向き。まずXで反応を見るのも手
収益 80 → 有料記事・商品化まで育てる価値がある

並び替えメニューで「収益化期待順」などに切り替えると、狙いに合うニュースが上に来ます。なおスコアはキーワードと情報源の信頼度による機械的な目安で、絶対的な評価ではありません。

うまくいかないとき

  • 症状コピーを押しても反応しない → ファイル直開きだとブラウザがコピーを止めることがあります。枠内の文字を長押し→すべて選択→コピーで代用できます
  • 症状文字が枠からはみ出る・真っ白になる → 古いファイルを開いています。最新版をダウンロードし直し、開いたら一度再読み込みしてください
  • 症状ニュースが増えない → プレビュー版の仕様です（デモ8件固定）。本物のニュースはGitHub公開後に1日3回自動で入ります

本番公開すると何が変わるか

  • 追加本物のAIニュースが約20ソースから毎日3回（朝7時・昼12時・夜19時）自動で入る
  • 追加スマホからいつでもURLで開ける（ブックマークで毎朝チェック→その場で記事化）
  • 同じ操作方法・タブ・プロンプトの使い方はプレビュー版と完全に同じ。今の練習がそのまま活きます
